# 示例科技有限公司 网络管理规范

> 依据《网络安全法》第二十一条（网络安全等级保护制度）及第二十五条（网络运行安全）制定。

## 第一章 网络架构

公司网络按"核心-汇聚-接入"三层架构部署。总部核心交换机采用双机热备（H3C S10500 系列），各楼层通过千兆光纤汇聚到核心层。互联网出口带宽 500Mbps，电信为主链路、联通为备用链路，通过 BGP 实现双链路自动切换。

办公区按工位部署双口网线面板：蓝色端口接入内网（10.10.0.0/16），白色端口接入外网（用于访客或测试设备）。会议室、茶水间、接待区部署企业级 WiFi 6（802.11ax）AP，信号覆盖不低于 -67dBm。

服务器区（DMZ）与办公内网通过防火墙隔离，仅开放经审批的服务端口。开发测试环境与生产环境物理隔离，测试环境不允许访问生产数据。

## 第二章 IP 地址规划

内网 IP 地址段为 10.0.0.0/8，按部门划分 VLAN：

| VLAN ID | 网段 | 部门 | 说明 |
|---------|------|------|------|
| 10 | 10.10.0.0/24 | 管理层 | 包含 C-level 办公室 |
| 20 | 10.10.1.0/24 | 研发部 | 含开发服务器 |
| 30 | 10.10.2.0/24 | 产品与设计部 | |
| 40 | 10.10.3.0/24 | 市场与销售部 | |
| 50 | 10.10.4.0/24 | HR 与行政部 | |
| 60 | 10.10.5.0/24 | 财务部 | 财务 VLAN 额外 ACL 隔离 |
| 100 | 10.10.100.0/24 | 服务器区 | DMZ，严格访问控制 |
| 200 | 10.10.200.0/24 | 设备管理 | 网络设备带外管理 |

DHCP 服务器为员工设备自动分配 IP，租约期 8 小时。服务器、网络设备和打印机使用静态 IP，须在 IT 部门 IP 地址管理系统（phpIPAM）中登记方可使用。新设备接入网络前须向 IT 部门提交网络准入申请，提供设备 MAC 地址、设备类型和所属部门。IT 部门在 4 个工作小时内完成审批和 VLAN 配置。

## 第三章 无线网络管理

公司办公 WiFi SSID 为 ExampleTech-Office（5GHz 频段优先），采用 WPA3-Enterprise 加密和 802.1X 认证。员工凭域账号和密码登录，客户端证书由公司 PKI 自动下发。访客 WiFi SSID 为 ExampleTech-Guest（2.4GHz），采用 WPA2-Personal 加密，密码由前台每日更新并公示于接待台。访客账号带宽限制为 10Mbps/设备，办公 WiFi 不设限。访客网络通过独立 VLAN 和 ACL 与内网完全隔离，仅能访问互联网。

严禁任何员工在工位私自架设无线路由器、随身 WiFi 或开启手机热点。IT 部门通过无线入侵检测系统（WIDS）7×24 监控非法 AP。发现非法 AP 后，IT 部门有权立即断网并通报部门负责人，违规者取消 VPN 权限 30 天。

## 第四章 VPN 远程接入

公司部署 SSL VPN 网关（深信服 SSL VPN）用于远程办公。VPN 接入须使用域账号 + 动态令牌（硬件 Token 或手机 App 生成的一次性密码），实现双因素认证。VPN 客户端支持 Windows 10/11、macOS 12+、iOS 16+ 和 Android 13+。

VPN 会话策略：单次会话最长 12 小时，空闲 30 分钟自动断开。同一账号同时仅允许一个活跃会话。离职员工 VPN 权限随域账号同步注销。IT 部门每月审计 VPN 登录日志，异常登录（异地 IP、非工作时间大量数据传输等）触发安全告警。

## 第五章 上网行为管理

公司部署上网行为管理设备（深信服 AC），对互联网访问进行日志记录和策略控制。禁止在工作时间（9:00-18:00）访问视频流媒体、网络游戏、博彩、色情、比特币挖矿等与工作无关的网站和应用。P2P 下载工具（BT、电驴、迅雷等）全天禁止使用。应用商店和软件更新流量在工作时间限速 5Mbps。

上网日志保存不少于 6 个月。日志仅在以下情况下经审批后调取：安全事件调查、合规审计、法律取证。IT 部门每月生成上网行为统计报告（脱敏至部门级），各部门负责人可查看本部门员工的总体上网行为概况。

## 第六章 网络故障管理

网络故障按影响范围和紧急程度分三级：
- 一级（P0）：全网或核心业务系统中断，影响≥50% 员工，15 分钟内响应，2 小时内恢复；
- 二级（P1）：单楼层或单部门网络中断，30 分钟内响应，4 小时内恢复；
- 三级（P2）：个别用户网络故障，1 小时内响应，8 小时内恢复。

员工报告网络故障通过 IT 服务台（分机 8888 或 OA 工单系统）。IT 部门建立网络故障知识库，常见故障提供自助排查指南。每季度对网络故障进行分析统计，对重复发生的同类型故障制定根治方案。

## 第七章 网络变更管理

网络变更操作（设备上线/下线、配置修改、版本升级、链路割接等）须提前提交变更方案，内容包括变更目的、影响范围、操作步骤、回滚方案和验证方法。变更方案经 IT 负责人审批后方可执行。标准变更窗口为每周四 20:00-23:00 或周六 9:00-18:00（选择影响最小的时间段）。紧急变更（安全漏洞修复、故障抢修）可先执行后补单，但须在变更后 24 小时内补交变更记录。所有变更操作须在变更日志中记录操作人、时间、内容和结果，保存不少于 1 年。

## 第八章 附则

本规范由 IT 部负责解释和修订。规范每年至少审视一次，或在发生重大网络安全事件后 30 日内完成修订。本规范自发布之日起施行。
