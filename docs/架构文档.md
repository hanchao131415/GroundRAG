# 项目一 架构文档

## 数据流图

```
┌─────────────────────────────────────────────────────────────────┐
│                        ① 数据接入层                                │
│  PDF/DOCX/MD ──→ PyMuPDF/python-docx ──→ 页面级原始文本          │
│  PDF 扫描件: PyMuPDF 无文字层时 → OCR 兜底(Tesseract 300dpi 中英) │
│       ↓                                                          │
│  _enhance_metadata(): 提取 source/department/page 元数据          │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                        ② 文档处理层                                │
│  _split_text_and_tables()                                        │
│    ├── 识别表格边界（|...|）→ 表格块整体不切                      │
│    └── 文本块 → RecursiveCharacterTextSplitter(500字+50overlap)  │
│       ↓                                                           │
│  每块打 chunk_id / parent_id / block_type / chunk_index           │
│  parent_child_map: child_id → parent_id（父子检索）               │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                        ③ 索引层                                   │
│  BGE embedding (normalize=True) → FAISS(IndexFlatIP, = cosine)   │
│  BM25Retriever (jieba 中文分词 + 40+ 英文技术词注册)              │
│  增量重建：检测 content_hash 变化 → 全量重建(防僵尸向量)          │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                        ④ 检索层（运行时）                          │
│  User Query                                                      │
│    ├── ① 缓存查询（精确哈希 / 语义cosine，带权限签名）             │
│    │      └── 命中 → 0.1ms 直接返回                               │
│    ├── ② 短路路由（明确问题跳过LLM）→ intent=retrieval(0ms)       │
│    ├── ③ 查询改写（模糊问题走LLM ainvoke）                        │
│    ├── ④ RBAC 权限过滤（先过滤 chunks 子集，再检索）               │
│    ├── ⑤ 双路召回                                                │
│    │      ├── 向量 IP/cosine（FAISS.similarity_search_with_score） │
│    │      └── BM25（rank_bm25 + jieba）                           │
│    ├── ⑥ RRF 融合（1/(k+rank)，k=60，chunk_id 去重）              │
│    ├── ⑦ MMR 去重（content_hash 精确去重 + 3-gram Jaccard 多样性）  │
│    └── ⑧ BGE-Reranker 精排（Cross-Encoder，按分数截断topK）       │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                  ⑤ 生成层（带 LLM 降级链）                         │
│  _build_context(): 拼接检索结果，max_length=3000 截断              │
│  防幻觉Prompt: 只能据上下文回答，标注[文档X]，无依据说"未找到"      │
│  FallbackLLM.invoke(): primary→fallback 自动降级(熔断冷却60s)      │
│  ↓                                                               │
│  回答（带溯源）+ 存缓存（带权限签名）                               │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                   ⑥ 可观测层（全程）                                │
│  本地 Trace(JSONL): 每步耗时 / 输入输出 / token / 成本(按provider) │
│  token 提取: 5 种格式兼容 + 流式字符估算 + DeepSeek stream_usage  │
│  Langfuse(云端): CallbackHandler → 可视化链路 + 成本统计          │
│  评测: run_eval.py(三维度) / ab_compare.py(A/B对比)              │
│  CI: GitHub Actions(pytest 19项) / Locust 压测                   │
└─────────────────────────────────────────────────────────────────┘
```

## API 分层（认证边界）

```
                    ┌──────────────────────┐
   客户端 ──Bearer──▶│  /auth/login         │ ──签发 JWT(user_id)
                    │  (无需认证)          │
                    └──────────────────────┘
                              │ token
                              ▼
                    ┌──────────────────────┐
                    │  /api/v1/chat        │── Depends(get_current_user)
                    │  /api/v1/search      │── 依赖注入校验 JWT
                    │  /api/v1/stats       │── user_id 只从 JWT 解码
                    └──────────────────────┘
                              │
              ┌───────────────┴───────────────┐
              ▼                               ▼
    ┌──────────────────┐              ┌──────────────────┐
    │ /health(免认证)  │              │ RBAC 授权层      │
    │ docker探针/监控  │              │ user→departments │
    └──────────────────┘              └──────────────────┘
```

**关键设计**：user_id 从 JWT 解码，不是客户端请求体 → 客户端伪造 `{"user_id":"admin"}` 无效。三层安全：认证(JWT) → 授权(user_service) → 过滤(retrieval)。

## 模块职责

| 模块 | 文件 | 职责 | 对应《真实RAG全貌》子系统 |
|---|---|---|---|
| LLM 工厂 | `llm_factory.py` | 协议适配(OpenAI/Anthropic/Gemini) + 供应商预设 + 重试 | ④ |
| **LLM 降级链** | `llm_fallback.py` | **FallbackLLM: primary挂→自动切备用,熔断冷却60s** | ④ |
| 数据准备 | `data_preparation.py` | 多格式加载、元数据提取、表格感知切分、父子检索 | ①② |

| 检索优化 | `retrieval_optimization.py` | 混合检索+RRF+BGE-Reranker+RBAC权限过滤 | ③⑤ |
| 生成集成 | `generation_integration.py` | 防幻觉Prompt+生成+查询改写+意图路由+异步支持 | ④ |

## 技术决策记录

| 决策 | 选型 | 为什么不选别的 |
|---|---|---|
| 向量检索 | FAISS IndexFlatIP | 归一化向量下=cosine,数值直观(0~1);L2 距离范围0~2要换算 |
| 关键词检索 | BM25(rank_bm25+jieba) | 纯Python零运维;ES等规模化才需要 |
| 精排 | BGE-Reranker(cross-encoder) | 比LLM-as-Reranker快100倍;解决了向量距离重叠问题 |
| 缓存 | 精确(hash)+语义(cosine) | 四重校验防误命中;权限签名隔离 |
| 可观测 | Langfuse+本地Trace | Langfuse可视化,本地Trace即时调试;双保险 |
| Python版本 | 3.12 | 3.14太新,AI生态未适配(ragas/FlagEmbedding编译失败) |
| transformers | 4.46.3 | 5.x移除了prepare_for_model,BGE-Reranker报错 |
| **认证** | **JWT(HS256) + PyJWT** | **无状态(水平扩容不共享session);不选python-jose因它需额外加密依赖** |
| **降级链** | **组合模式(非继承)** | **duck-type兼容LangChain Runnable;不继承RunnableLambda因拿不到usage_metadata** |
| **熔断策略** | **冷却时间(60s)** | **比健康检查简单;失败的provider冷却后再试,避免每请求都白试primary** |
| **CI** | **GitHub Actions** | **免费for public repo;不依赖真实LLM(fake key跑纯逻辑测试)** |
| **压测** | **Locust(Python)** | **可写Python脚本模拟用户行为;wrk只测HTTP不够灵活** |
